Ejercicio # 2.

  • Del proyecto anterior vas a implementar un procedimiento que se llame eficiencia_developer, que reciba como argumento de entrada el id del desarrollador y que contenga como argumento de salida el id de las tareas no completadas. Deberá regresar algo asi como "El siguiente desarrollador tiene por completar las las tareas cuyos id' son: 1,6, y 7".

  • Implementar otro procedimiento almacenado cuyo nombre sea analziar_tarea_incompleta; cuyos argumentos de entrada sean el id del desarrollador y un id de alguna tarea incompleta( que se puede obtener previamente del procedimiento anterior). El procedimiento debe tener un tipo de salida que indique el porcentaje y la fecha última de actualizacion de dicha tarea.

  • Implementar un procedimiento almacenado que se llame eficiencia_empresa, que solo contenga un tipó de salida como argumento, el cual debe indicar: El procentaje de tareas no completadas de todos los desarrolladores. Sacar el procentaje con base al numero de tareas totales por todos los desarrolladores. NOTA: Es importante usar un cursor para este procedimiento.

Ejercicio 1. Fecha de entrega: Sabado 27 de Septiembre de 2014

Implementa el siguiente esquema de datos

  • Crea las tablas
  • Para cada tabla vas a crear una secuencia
  • Para poder hacer una inserción de registro generar un procedimiento almacenado para cada tabla
  • Completar la tabla reparto.
